# Состояние проекта

> Обновлено: 2026-04-25

## Что это за проект

`Домфиндом` — web-приложение для личной и семейной домашней бухгалтерии.

Текущий активный продукт:
- `backend/` — FastAPI API;
- `frontend/` — React + TypeScript + Vite;
- `backend/auth/` — авторизация, семьи, письма, сессии, предпочтения пользователя.

Desktop-часть исторически осталась в репозитории, но не является главным направлением продукта.

## Что уже работает

### Авторизация и аккаунт

- регистрация;
- вход и выход;
- подтверждение email;
- восстановление пароля по email;
- удаление аккаунта через подтверждение по email;
- личные настройки профиля;
- `display_name` для показа имени вместо email;
- темы `light`, `dark`, `system`;
- журнал событий и активные сессии;
- резервная копия пользовательских данных.

### Финансовые функции

- главная страница с балансом и обзором месяца;
- доходы и расходы;
- категории;
- планирование;
- счета и капитал;
- переводы;
- быстрый ввод операций;
- повторяющиеся и плановые операции.

### Семейный режим

- переключение между личным и совместным режимом;
- создание семьи;
- приглашения в семью;
- роли `owner`, `member`, `viewer`;
- семейная сводка;
- семейная история операций на странице `Транзакции`;
- семейный статус бюджетов, видимый участникам семьи;
- аудит категорий семьи для владельца;
- preview и подтверждение связей категорий владельцем семьи;
- автор операций показывается по имени профиля, а не по email, если имя задано;
- вкладка `Семья` доступна в совместном режиме даже до создания семьи.

### Мобильная версия

- отдельная нижняя навигация на телефоне;
- вкладки в одну линию с горизонтальной прокруткой;
- верхняя зона на телефоне оставляет уведомления и аккаунт;
- случайное масштабирование страницы на телефоне отключено;
- семейные мобильные карточки отдельно дополированы.

## Важные продуктовые правила

- Основной интерфейс проекта — web.
- `display_name` имеет приоритет над email в UI.
- Личный и семейный режимы нельзя смешивать случайно.
- Семейные данные — это обзор поверх данных участников, а не физическое слияние всех личных записей в одну общую базу.
- Обычный участник семьи видит семейные агрегаты и лимиты, но не видит owner-only управление приглашениями и аудитом категорий.
- Владелец семьи отвечает за подтверждение связей категорий и решения аудита.
- Опасные действия должны быть понятными и подтверждаемыми.
- Пользовательский текст должен быть на русском языке.

## Текущая схема хранения данных

- `auth.db` хранит пользователей, семьи, роли, сессии, токены и предпочтения.
- `auth.db` также хранит семейные смысловые категории и подтвержденные связи локальных категорий с семейным смыслом.
- Финансовые данные хранятся в отдельных пользовательских SQLite-базах.
- Семейный обзор строится агрегированием по участникам.

Это важно:
- при семейных расчетах всегда нужно помнить про per-user БД;
- кэш нельзя наивно переиспользовать между базами разных участников.

## Production

Текущий production размещен на `https://domfindom.ru`.

Схема:
- backend-репозиторий на REG.RU shared hosting;
- backend запускается через `Passenger`;
- frontend собирается локально;
- содержимое `frontend/dist` выкладывается как статика на сайт.

Проверочный endpoint:

```bash
curl -L https://domfindom.ru/api/v1/health
```

Ожидаемый ответ:

```json
{"status":"ok"}
```
