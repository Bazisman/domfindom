## Что Это За Проект

`Домашняя бухгалтерия` — сейчас desktop-приложение на Python + CustomTkinter + SQLite для учёта личных финансов, но стратегически проект разворачивается в сторону универсального web-приложения для ПК и телефона.

Ключевые сущности:
- фактические транзакции доходов и расходов
- плановые транзакции (`status='planned'`)
- категории
- бюджеты
- основной счёт
- счета капитала
- переводы между счетами
- сверка баланса

---

## Архитектура

- `views/` — интерфейс
- `services/` — бизнес-логика и уведомления UI
- `core.py` — SQL, БД, ключевые расчёты
- `models.py` — dataclass-модели
- `finance.db` — рабочая SQLite БД

Главный принцип: UI не должен сам придумывать финансовую логику, она должна жить в `services/` и `core.py`.

---

## Что Уже Работает

- учёт доходов и расходов
- автoотчисления в капитал
- несколько счетов капитала
- переводы между счетами
- бюджеты по категориям
- регулярные шаблоны и плановые транзакции
- автоматическое исполнение просроченных плановых транзакций
- сверка реального баланса
- резервные копии и логи

---

## Текущий Фокус

Сейчас проект в трёх параллельных рабочих потоках:
- стабилизация логики планирования
- поэтапная оптимизация текущего desktop UI как временного рабочего клиента
- подготовка архитектурного перехода к web-версии

Особенно важно:
- не смешивать `planned` и `actual` в фактических расчётах
- не дублировать влияние одной и той же траты в прогнозах
- держать понятную связь между бюджетами, плановыми расходами и реальным балансом
- сокращать полные перерисовки интерфейса там, где можно обновить только данные или видимую часть экрана
- не вкладываться в desktop UI так глубоко, чтобы это мешало будущей web-миграции
