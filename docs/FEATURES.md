# 🔑 Ключевые функции

## Баланс как состояние

Баланс не пересчитывается при запуске. Он:
- Хранится в таблицах `accounts` и `capital_accounts`
- Обновляется мгновенно при операциях
- Является материализованным представлением

Это даёт: быструю загрузку, высокую производительность, надёжность.

---

## Отображение баланса

В верхней части приложения отображается:
- **Месяц и год** — текущий период
- **Баланс** — остаток на основном счёте
- **Доход/Расход/Вклад** — суммы за текущий месяц

```
Март 2026
────────────────
37 882.20 ₽
Доход: 50 000 ₽ | Расход: 30 000 ₽ | Вклад: 5 000 ₽
```

Данные обновляются в реальном времени при каждом изменении транзакций.

## 1. Автоотчисления в капитал

### Логика

- При добавлении дохода (income) срабатывает автоотчисление
- Процент отчисления (по умолчанию 10%) настраивается во вкладке "Капитал"
- Отчисления идут на **основной счёт капитала** (где `is_default = 1`)
- Основной счёт получает: `доход - отчисление`
- Счёт капитала получает: `доход * процент / 100`

### Схема работы

```
Доход 10,000 руб
│
▼ Автоотчисление 10%
│
▼                  ▼
Основной счёт     Счёт капитала
(accounts, id=1)  (is_default=1)
│                  │
▼                  ▼
+9,000 руб        +1,000 руб
```

### Настройка

Процент отчисления настраивается во вкладке "Капитал":
- Можно включить/отключить автоотчисления
- Задать процент (0-100%)

### Исключения

Категория **"Остаток"** не облагается отчислением в капитал. Это позволяет вносить начальный баланс без создания отчислений.

## 2. Управление капиталом

### Возможности

- Создание нескольких счетов капитала (например, "ТБанк", "Сбер")
- Один счёт помечается как основной (⭐)
- Все автоотчисления идут на основной счёт
- Переводы между счетами
- История переводов

### Операции

- **Добавить счёт** — название, начальный баланс, иконка, цвет
- **Редактировать** — изменить название, баланс, иконку, цвет
- **Удалить** — деактивировать счёт
- **Сделать основным** — установить как счёт для отчислений

---

## 3. Транзакции

### Типы

- **Доход** — приход денег
- **Расход** — траты

### Функции

- Добавление транзакций (сумма, категория, дата, комментарий)
- Редактирование транзакций
- Удаление транзакций (с откатом автоотчислений для доходов)
- Фильтрация по периоду:
  - Всё время
  - Текущий месяц
  - Прошлый месяц
  - Текущий год
  - Выбрать даты
- Фильтрация по категории
- Экспорт в CSV

---

## 4. Категории

### Типы категорий

- **Доход** — только для доходов
- **Расход** — только для расходов
- **Оба** — и для доходов, и для расходов

### Управление

- Создание категорий
- Редактирование (название, иконка, цвет)
- Удаление (мягкое — is_active = 0)

---

## 5. Бюджеты

### Возможности

- Установка бюджета на месяц или год
- Предупреждение при превышении бюджета

### Логика

При добавлении расхода:
1. Проверяется бюджет категории
2. Если превышено — показывается предупреждение
3. Пользователь решает: добавить или отменить

## 6. Обновление интерфейса

### Механизм уведомлений

```python
# Сервис хранит список слушателей
self._listeners = []

# Вкладка подписывается
self.transaction_service.add_listener(self.on_data_changed)

# При изменении данных — уведомляются все слушатели
def notify_listeners(self):
    for listener in self._listeners:
        listener()
```

## 7. Логирование

### Настройки

- Ротация логов: 10 MB на файл, 5 бэкапов
- Отдельный файл для ошибок: 5 MB, 3 бэкапа
- Кодировка: UTF-8 (корректное отображение в Windows)
- Вывод в консоль и файл

### Использование

```python
from utils.logger import app_logger

app_logger.info("Сообщение")
app_logger.error("Ошибка", exc_info=True)
app_logger.debug("Отладка")
```
