# Session Addendum 2026-04-11 (Encoding Incident)

## Incident
- User-facing auth UI showed mojibake symbols instead of readable text.
- Root cause: mixed/incorrect text encoding edits in frontend source files.

## Recovery Actions
- Restored readable UI text in:
  - `frontend/index.html`
  - `frontend/src/pages/LoginPage.tsx`
  - `frontend/src/pages/SecurityPage.tsx`
  - `frontend/src/AppShellNext.tsx`
- Rebuilt frontend and redeployed static assets.
- Verified deployed files on server by hash and by content checks.

## Permanent Prevention Added
- Added repository-wide encoding guardrails:
  - `.editorconfig` (`charset=utf-8`, `end_of_line=lf`)
  - `.gitattributes` (`text eol=lf` rules for code/docs)
- Added automated encoding validator:
  - `tools/check_encoding.py`
- Added policy and protocol:
  - `docs/ENCODING_POLICY.md`

## Mandatory New Rule
- Before any deploy: run `python tools/check_encoding.py`.
- If it fails, deployment is blocked until encoding is fixed.
