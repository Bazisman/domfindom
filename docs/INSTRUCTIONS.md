# 📜 Инструкции для ИИ-ассистента

> **Версия:** 1.0
> **Дата:** 5 апреля 2026

Этот документ содержит инструкции для ИИ-ассистента, работающего над проектом "Домашняя бухгалтерия".

---

## 1. Общие принципы работы

### 1.1. Цель

ИИ ассистент — совместный разработчик, который:
- Помогает писать и рефакторить код
- Документирует изменения
- Учится на ошибках
- Понимает архитектуру проекта

### 1.2. Подход к редактированию

ИИ **не переписывает файлы полностью**, а:
- Использует `str_replace` для изменения **конкретных блоков**
- Анализирует контекст перед внесением изменений
- Вносит минимально необходимые изменения
- Сохраняет структуру и стиль документа

### 1.3. Безопасность

Перед любым изменением:
- Проверяется синтаксис: `python -c "from X import Y"`
- Читается файл перед редактированием (`read_file`)
- Проверяется, что заменяемый текст **уникален**

## 2. Работа с документацией

### 2.1. Формат документов

Все документы — `.md` (Markdown). Используется:
- Заголовки `#`, `##`, `###`
- Таблицы `|---|---|`
- Блоки кода ```python
- Списки `-` и `1.`

### 2.2. Документы и их назначение

| Файл | Назначение |
|------|------------|
| `KODA.md` | Общее описание проекта, структура, запуск |
| `ARCHITECTURE.md` | Архитектура, слои, схема БД |
| `API.md` | Методы сервисов, модели данных |
| `CODE_STYLE.md` | Именование, стиль кода, паттерны GUI |
| `CHANGELOG.md` | История изменений, версии |
| `ERROR_LEARNING.md` | Уроки ошибок, грабли |
| `INSTRUCTIONS.md` | Инструкции для ИИ-ассистента (этот файл) |
| `AI_GUIDE.md` | Руководство для ИИ: архитектура, константы, операции |

### 2.3. Алгоритм обновления документации

1. **Прочитать** все связанные документы
2. **Найти** сегмент, который нужно изменить
3. **Использовать `str_replace`**, чтобы заменить только нужный блок
4. **Не добавлять** лишний текст
5. **Сохранить** стиль и форматирование

### 2.4. Маркер завершения сессии

Если пользователь пишет:

`конец работы`

это команда завершить рабочую сессию по правилам.

После этого ИИ обязан:
1. Обновить `TASKS.md`, если изменился статус задач
2. Обновить `docs/PROJECT_STATE.md`, если изменился текущий фокус или состояние логики
3. Обновить `docs/DECISIONS.md`, если принято новое правило или договорённость
4. Обновить `docs/ERROR_LEARNING.md`, если выявлена новая ошибка или анти-паттерн
5. Обновить текущий файл в `docs/sessions/`
6. Обновить `docs/CHANGELOG.md`, если было заметное изменение функциональности

Фраза `конец работы` означает не просто остановку ответа, а корректное закрытие сессии с документацией.

### 2.5. Маркер промежуточного протоколирования

Если пользователь пишет:

`запротоколируй`

это команда сделать промежуточное сохранение состояния проекта.

После этого ИИ должен:
1. Обновить `TASKS.md`
2. Обновить `docs/PROJECT_STATE.md`
3. Обновить `docs/DECISIONS.md`, если появилась новая договорённость
4. Обновить текущий файл в `docs/sessions/`
5. Обновить `docs/ERROR_LEARNING.md`, если была выявлена новая ошибка

Фраза `запротоколируй` не означает конец работы. После неё сессию можно продолжать.

## 3. Работа с базой данных

### 3.1. Структура БД

#### Таблицы

- `accounts` — основной счёт (id=1)
- `capital_accounts` — счета капитала (id >= 100)
- `transactions` — транзакции
- `recurring_templates` — шаблоны регулярных платежей
- `transfers` — переводы

#### Поля в `transactions`


| Поле | Описание |
|------|----------|
| `status` | 'actual' или 'planned' |
| `executed_at` | дата исполнения (NULL для planned) |
| `template_id` | ссылка на шаблон |

### 3.2. Миграции

- Новые поля и таблицы добавляются через `_migrate_recurring_transactions()`
- Всегда проверять, существует ли поле (`try-except`)
- Создавать индексы для новых полей

## 4. Работа с кодом

### 4.1. Архитектура

```
┌─────────────────────────────────────────────────────────┐
│ GUI (views/)                                            │
│ Пользовательский интерфейс, вкладки                     │
├─────────────────────────────────────────────────────────┤
│ Services (services/)                                    │
│ Бизнес-логика, подписки                                 │
├─────────────────────────────────────────────────────────┤
│ Core (core.py)                                          │
│ Работа с БД, SQL-запросы                                │
├─────────────────────────────────────────────────────────┤
│ Database (SQLite)                                       │
│ finance.db                                              │
└─────────────────────────────────────────────────────────┘
```

### 4.2. Паттерны

#### Observer (Слушатель)

```python
self.transaction_service.add_listener(self.on_data_changed)

# Обновление при изменении данных
def on_data_changed(self):
    self.refresh()
```

#### Сервисный слой
- `TransactionService` — транзакции, бюджеты, планирование
- `CategoryService` — категории

#### DAO
- `core.py` содержит все SQL-запросы
- Все операции с БД через транзакции
