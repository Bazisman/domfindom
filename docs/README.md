# Документация проекта

> Обновлено: 2026-04-27

Эта папка теперь разделена на две части:

- живые документы, по которым ведется текущая работа;
- архив, где лежат старые планы, промежуточные протоколы и исторические материалы.

- `MONEY_MODEL_STATUS.md` — текущая модель денег: что уже сделано в `Деньгах`, семье, подушке, инвестициях и что еще осталось.

## Архив

Старые документы перенесены в `docs/archive/`.

Там лежат:
- старые планы миграции и внедрения;
- подробные session-файлы;
- legacy-документы desktop-эпохи;
- промежуточные протоколы, которые больше не нужны как ежедневный рабочий слой.

В активном слое больше не должно быть `docs/sessions/`, старых планов внедрения и legacy-чеклистов. Если такой файл появляется рядом с живыми документами, сначала проверить, не должен ли он лежать в `docs/archive/`.

Важно:
- архив не удаляет историю;
- перед возвратом к старым решениям можно поднимать материалы из архива;
- новые ежедневные протоколы в архив больше не складываются по умолчанию.

## Правило протоколирования

Если пользователь пишет `запротоколируй`, в первую очередь обновляется `docs/WORK_LOG.md`.

Отдельный файл в архивном стиле нужен только если:
- произошел большой инцидент;
- нужен развернутый технический postmortem;
- нужно сохранить большую самостоятельную веху отдельно от общего журнала.
