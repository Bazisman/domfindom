# История проекта

> Обновлено: 2026-04-18
> Формат: краткая история крупных этапов, без мелкого шума.

## Этап 1. Desktop-проект

Изначально проект жил как desktop-приложение на `CustomTkinter` с SQLite и единым локальным контуром.

Что перешло в web-эпоху из этой стадии:
- базовая финансовая логика;
- модель счетов, категорий, бюджетов, капитала;
- часть тестов и core-слой;
- накопленные продуктовые договоренности по домашней бухгалтерии.

## Этап 2. Переход к web

Проект начал переход с desktop на web-модель:
- backend на `FastAPI`;
- frontend на `React + TypeScript + Vite`;
- production-размещение на `domfindom.ru`.

На этом этапе были приняты ключевые решения:
- считать web-версию главным продуктом;
- desktop не удалять, но не использовать как ориентир для новых продуктовых решений;
- хранить документацию как часть рабочей памяти проекта.

## Этап 3. Авторизация и персонализация

В проект были добавлены:
- регистрация и вход;
- cookie-сессии;
- подтверждение email;
- восстановление пароля;
- удаление аккаунта по email-подтверждению;
- личные настройки и `display_name`;
- журнал событий и активные сессии.

Этот этап перевел проект из single-user утилиты в пользовательский web-сервис.

## Этап 4. Семейный режим

Была спроектирована и внедрена семейная модель:
- семья — это не слияние личных данных;
- у каждого участника своя учетная запись и свои данные;
- поверх них строится семейный обзор;
- добавлены приглашения, роли и семейная лента операций.

Ключевые продуктовые решения:
- роли сведены к `owner`, `member`, `viewer`;
- в UI для семейных сценариев показывается имя профиля, а не email;
- вкладка `Семья` доступна уже в совместном режиме, даже до фактического создания семьи.

## Этап 5. Безопасность и стабильность production

Были усилены:
- CSRF-защита;
- secure headers;
- ограничения на rate limit;
- правила session secret;
- защита от ошибок Python-совместимости на production;
- контроль email-flow на реальном хосте.

## Этап 6. Борьба с кодировкой

Проект пережил несколько инцидентов с битой кириллицей.

После этого были добавлены:
- `.editorconfig`;
- `.gitattributes`;
- `.vscode/settings.json`;
- git hooks;
- `tools/check_encoding.py`;
- отдельные правила для PowerShell и UTF-8 без BOM.

С этого момента кодировка стала отдельным критическим инвариантом проекта.

## Этап 7. Мобильная polish-фаза

В мобильной версии были доведены:
- нижняя навигация;
- прокручиваемые вкладки в одну линию;
- layout карточек семьи;
- мобильное меню аккаунта;
- защита от случайного zoom.

## Этап 8. Переход на новый режим документации

Старый подход с множеством мелких session-файлов перестал быть удобным.

Новая схема:
- `WORK_LOG.md` — один живой журнал;
- `WORKING_MEMORY.md` — накопленные устойчивые выводы;
- `archive/` — архив старых планов, протоколов и legacy-документов.

## Последняя важная техническая правка

2026-04-18:
