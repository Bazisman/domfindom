# Старт Нового Чата

> Обновлено: 2026-04-19
> Назначение: короткая команда для старта новой рабочей сессии без потери проектного регламента.

## Короткая команда

Если пользователь в новом чате пишет только:

`начинаем`

это означает следующий обязательный протокол старта:

1. Прочитать `TASKS.md`.
2. Прочитать `docs/PROJECT_STATE.md`.
3. Прочитать `docs/DECISIONS.md`.
4. Прочитать `docs/WORKING_MEMORY.md`.
5. Прочитать последние записи в `docs/WORK_LOG.md`.
6. При необходимости прочитать `docs/PROJECT_HISTORY.md`.
7. Не просто просмотреть эти файлы, а извлечь из них:
   - что сейчас происходит в проекте;
   - какие правила работы обязательны;
   - как в этом репозитории нужно делать правки, git и деплой;
   - какие антиошибки уже накоплены и не должны повторяться.
   - какие последние процессные выводы нужно применить сразу: сначала безопасное восстановление после регрессии, потом точечная правка, потом проверка кодировки, сборки и только затем выкладка.
8. Коротко сообщить пользователю, что понято по текущему состоянию проекта и по способу работы.
9. Только после этого переходить к коду, тестам, git-операциям или деплою.

Иными словами, команда `начинаем` означает не только чтение контекста проекта, но и обязательное восстановление рабочего регламента из документации.

## Что не подразумевается автоматически

Команда `начинаем` сама по себе не означает:

- автоматический `git commit`;
- автоматический `git push`;
- автоматический деплой на production.

Эти действия выполняются только если пользователь явно попросил об этом отдельной фразой или в рамках той же задачи без двусмысленности.

## Короткая рекомендуемая форма для пользователя

Если нужен просто нормальный старт новой сессии, достаточно написать:

`начинаем`

Если нужен старт сразу с ожиданием релиза, лучше написать:

`начинаем. после выполнения покажи статус локальных изменений и не делай git/deploy без отдельной команды`

или:

`начинаем. после выполнения подготовь изменения к коммиту, но не пушь и не деплой без моего подтверждения`
