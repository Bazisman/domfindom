# Журнал работ

> Обновлено: 2026-04-26
> Назначение: один общий живой протокол значимых изменений, проверок и выкладок.

## Правила

- Вести один общий журнал вместо нового файла на каждый мелкий шаг.
- Писать коротко: сначала результат для продукта или процесса, затем при необходимости одной строкой подтверждение проверки или релиза без списка команд и операционного лога.
- Если пользователь пишет `запротоколируй`, сначала обновлять этот файл.
- Отдельный архивный файл нужен только для редких больших инцидентов или самостоятельных крупных вех.

### 2026-04-24

- Зафиксировали целевую пользовательскую логику прогнозов для личного режима как отдельную бизнес-модель, чтобы не смешивать `текущий баланс` и `остаток на конец месяца`.
- `Текущий баланс` должен пониматься как фактические деньги на текущую дату: входящий остаток прошлых месяцев + доходы текущего месяца по текущую дату - исполненные расходы текущего месяца по текущую дату.
- `Доход за месяц` в прогнозе должен состоять из двух частей: уже полученные доходы текущего месяца и еще не исполненные `planned`-доходы до конца текущего месяца.
- `Расход за месяц` в прогнозе должен состоять из трех частей: уже потрачено на текущую дату, точные `planned`-расходы до конца месяца и бюджетный прогноз на остаток месяца.
- Для `daily` бюджета зафиксировали целевую формулу прогноза:
  - `v = a * b`, где `a` — дневной лимит, `b` — число дней в месяце;
  - `z` — уже потрачено по категории на текущую дату, включая сегодня;
  - `d = a * days_left` — сколько еще планируется потратить до конца месяца;
  - `g = v - (z + d)` — прогнозный остаток бюджета к концу месяца;
  - в будущие расходы месяца должен попадать именно `d`, а не весь месячный лимит и не вся сумма `z + d`, потому что `z` уже учтено в факте.
- Для семейного режима договорились использовать ту же модель, но агрегировать факт, planned и бюджетный прогноз по всем участникам семьи.
- Реализовали новую формулу month-end forecast в коде:
  - личный и семейный `projected_balance` теперь считаются как результат текущего месяца, а не как `текущий баланс на счетах - будущие расходы`;
  - `daily` бюджет в forecast добавляет только расход на оставшиеся дни месяца;
  - `monthly` бюджет резервирует только еще не израсходованную и не покрытую `planned`-операциями часть лимита;
  - для `daily` бюджета месячный эквивалент снова считается по фактическому числу дней текущего месяца, а не по фиксированным `30`.
