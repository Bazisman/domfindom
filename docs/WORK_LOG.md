# Журнал работ

> Обновлено: 2026-04-18
> Назначение: один общий живой протокол значимых изменений, проверок и выкладок.

## Правила

- Вести один общий журнал вместо нового файла на каждый мелкий шаг.
- Писать коротко: что изменили, зачем, что проверили, что осталось.
- Если пользователь пишет `запротоколируй`, сначала обновлять этот файл.
- Отдельный архивный файл нужен только для редких больших инцидентов или самостоятельных крупных вех.

- Для мобильного layout добавлена безопасная нижняя зона под контент и мобильные панели, чтобы вкладки, уведомления и кнопки не уходили под нижнее меню.

  - `python tools/check_encoding.py --root .`
  - `python -m unittest tests.test_web_api.WebApiTestCase.test_family_dashboard_sums_each_member_balance_without_cache_bleed -v`
  - `npm run build`
- Production проверен через health endpoint: `{"status":"ok"}`.

- Выявили причину ошибки `Method Not Allowed` при добавлении источника денег на `domfindom.ru`: frontend уже был выложен, а backend checkout на сервере оставался на старом коммите без reconciliation-роутов, поэтому для production нужен отдельный `git pull --ff-only` перед рестартом.

- После деплоя backend checkout на production обновлен до `ecf0e2c`, свежий `frontend/dist` перезалит на `domfindom.ru`, выполнен restart через `tmp/restart.txt`, health endpoint вернул `{"status":"ok"}`, а `POST /api/v1/reconciliation/sources` без авторизации ответил `401`, то есть live-маршрут уже существует и больше не упирается в `405`.
- Для web-верстки ввели общий базовый паттерн summary-плиток: теперь блоки из планирования, счетов и сверки денег опираются на один набор классов `summary-grid` / `summary-card` / `summary-card-main`, чтобы новые одинаковые карточки не рисовались заново отдельными стилями и не расходились по внешнему виду.
- На странице `Транзакции` большие панели перевели в единый stack-layout: `Сверка денег`, `Новая транзакция` и `История транзакций` теперь живут в одном сценарии ширины как большие карточки одного уровня, без разнобоя между полноширинной верхней панелью и нижней двухколоночной сеткой.
- На странице `Транзакции` переставили порядок больших панелей под более частый пользовательский сценарий: сначала `Новая транзакция`, затем `Сверка денег`, затем `История транзакций`.
- На главной странице карточку текущего баланса расширили до свайпаемого сценария для семейного режима: в режиме `Семья` внутри той же большой карточки появляется второй слайд с текущим балансом семьи, доходом и расходом семьи, переключаемый свайпом влево/вправо и точками-индикаторами.
- Для desktop-версии этой же карточки добавили drag-жест мышью: семейный слайд баланса на главной теперь можно захватывать курсором и тянуть влево/вправо так же, как свайпом на телефоне.
- Доработали drag-механику семейного слайда на desktop до живого перетягивания: во время захвата мышью snap временно отключается и карточка идет за курсором свободно, а после отпускания мягко доводится до ближайшего слайда.
- Дополнительно сгладили момент отпускания мыши: плавное дотягивание к ближайшему слайду теперь запускается после выхода трека из режима drag, поэтому переход не обрывается резко.
- Для самого трека семейной карточки баланса включили явную smooth-прокрутку и `scroll-snap-stop`, чтобы доезд до ближайшего слайда был визуально заметным, а не мгновенным.
- После этого перевели финальное дотягивание семейного слайда на управляемую JS-анимацию с увеличенной длительностью, чтобы эффект доезда после отпускания мыши читался заметно медленнее.
- Для desktop дополнительно усилили видимость доезда при маленьком остаточном ходе: если до целевого слайда остается слишком мало, анимация теперь делает короткий overshoot и только потом мягко возвращается в итоговую позицию.
- Перестроили семейный слайдер баланса на направленную циклическую механику: desktop-перетягивание теперь выбирает следующий экран по последнему направлению жеста, плавно долистывает к нему без двойного довода и не упирается в край благодаря кольцевому треку из клонов.
