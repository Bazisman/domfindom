# Журнал работ

> Обновлено: 2026-04-18
> Назначение: один общий живой протокол значимых изменений, проверок и выкладок.

## Правила

- Вести один общий журнал вместо нового файла на каждый мелкий шаг.
- Писать коротко: что изменили, зачем, что проверили, что осталось.
- Если пользователь пишет `запротоколируй`, сначала обновлять этот файл.
- Отдельный архивный файл нужен только для редких больших инцидентов или самостоятельных крупных вех.

## Записи

### 2026-04-18

- В web-версии реализована сверка денег по мотивам старой desktop-логики: отдельные источники фактических денег, итог, разница, история сверок и автоматическая корректирующая транзакция `Корректировка` при расхождении.
- Для сверки добавлен backend API и встроенная панель в верхней части страницы `Транзакции`, чтобы сценарий жил рядом с операциями, как и в старой версии.
- Проверено:
  - `python -m unittest tests.test_web_api.WebApiTestCase.test_reconciliation_flow_creates_adjustment_transaction -v`
  - `python tools/check_encoding.py --root .`
  - `npm run build`

- На мобильной версии нижняя навигация переведена в более жесткий app-bar сценарий: фиксирована по нижнему краю экрана на всю ширину, без эффекта отдельной плавающей карточки поверх страницы.
- Для мобильного layout добавлена безопасная нижняя зона под контент и мобильные панели, чтобы вкладки, уведомления и кнопки не уходили под нижнее меню.
- Фикс выложен на `domfindom.ru`: коммит отправлен в `origin/main`, свежий `frontend/dist` загружен на сервер, выполнен restart через `tmp/restart.txt`.
- Проверено:
  - `python tools/check_encoding.py --root .`
  - `npm run build`
  - `https://domfindom.ru/api/v1/health` -> `{"status":"ok"}`

- Откатили неправильное изменение, которое переносило семейный баг на главную страницу вместо исправления самой семейной сводки.
- Исправили реальную причину неверного семейного баланса: кэш `balance` мог протекать между базами разных участников семьи.
- Для семейной сводки включено принудительное чтение свежего баланса без повторного использования значения от другого участника.
- Добавлен регрессионный тест на точный сценарий: Максим `90000` + Настя `-20000` = семейный баланс `70000`.
- Полностью перестроена схема документации:

  - добавлен `PROJECT_HISTORY.md`;
  - старые session-файлы, планы и legacy-документы перенесены в `docs/archive/`.

  - `python tools/check_encoding.py --root .`
